## About Kone Academy

**Kone Academy** is a multidisciplinary technology and innovation collective dedicated to bridging the gap between academic excellence, cutting-edge software engineering, and practical industrial solutions. 

We empower learners, researchers, and builders of all levels by providing world-class education, professional consulting, and hands-on engineering experiences. Our ecosystem comprises four specialized divisions, each focusing on distinct aspects of innovation and technology education.

### Key Features

- 🎓 **Comprehensive Education**: From beginner to advanced levels
- 🔬 **Research Excellence**: Academic and scientific research support
- 💻 **Modern Development**: Current programming languages and frameworks
- 🛠️ **Practical Engineering**: Real-world IoT, 3D modeling, and design solutions
- 🌍 **Global Community**: Connecting innovators worldwide
- 📱 **Digital-First Approach**: Cloud-based solutions and modern web technologies

## 🚀 Getting Started

### Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (v16 or higher) - [Download](https://nodejs.org/)
- **npm** (v8 or higher) - Comes with Node.js
- **Git** - [Download](https://git-scm.com/)
- **A modern web browser** - Chrome, Firefox, Safari, or Edge

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/PhilipKone/Kone-Academy.git
   cd Kone-Academy
   ```

2. **Navigate to the landing page directory**
   ```bash
   cd kca-landing-page
   ```

3. **Install dependencies**
   ```bash
   npm install
   ```

4. **Set up environment variables**
   ```bash
   # Create a .env.local file in the kca-landing-page directory
   cp .env.example .env.local
   
   # Add your Firebase configuration and other secrets
   # (See Environment Variables section below)
   ```

### Environment Variables

Create a `.env.local` file in the `kca-landing-page/` directory:

```env
# Firebase Configuration
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_auth_domain
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_storage_bucket
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id
VITE_FIREBASE_MEASUREMENT_ID=your_measurement_id

# Application Configuration
VITE_APP_URL=http://localhost:5173
VITE_API_ENDPOINT=https://your-api-endpoint.com
```

## 💻 Development

### Running the Development Server

```bash
npm run dev
```

The application will be available at `http://localhost:5173/`

**Hot Module Replacement (HMR)** is enabled, so changes will reload automatically in your browser.

### Building for Production

```bash
npm run build
```

This creates an optimized production build in the `dist/` directory.

### Preview Production Build

```bash
npm run preview
```

Test the production build locally before deployment.

### Available Scripts

| Command | Description |
|---------|-------------|
| `npm run dev` | Start the development server with HMR |
| `npm run build` | Build optimized production bundle |
| `npm run preview` | Preview production build locally |
| `npm run deploy` | Deploy to GitHub Pages |
| `npm run predeploy` | Pre-deployment build preparation |

## 🌐 Deployment

### GitHub Pages Deployment

The project is configured for automatic deployment to GitHub Pages using the `gh-pages` package.

**Manual Deployment:**

```bash
# Build and deploy in one command
npm run deploy
```

This will:
1. Build the optimized production bundle
2. Run `react-snap` for static site generation
3. Push the `dist/` folder to the `gh-pages` branch

**Automatic Deployment:**

GitHub Actions workflows can be set up to automatically deploy on pushes to the main branch.

### Environment Setup for Deployment

Ensure the following are configured:

1. **GitHub Pages Settings**: Set branch source to `gh-pages` in repository settings
2. **Firebase Hosting**: Configure Firebase for backend services (if needed)
3. **Custom Domain**: Point your domain (e.g., koneacademy.io) to GitHub Pages
4. **SSL/TLS**: Automatic with GitHub Pages or your domain provider

### Deploy to Firebase Hosting (Alternative)

```bash
# Install Firebase CLI globally
npm install -g firebase-tools

# Login to Firebase
firebase login

# Deploy
firebase deploy
```

## 🔧 Development Workflow

### Code Style & Standards

We follow modern JavaScript and React best practices:

- **ESLint**: For code quality (configure in `.eslintrc`)
- **Prettier**: For code formatting (configure in `.prettierrc`)
- **Component-Based Architecture**: Modular, reusable React components
- **Modern ES6+**: Arrow functions, destructuring, async/await

### Git Workflow

1. Create a feature branch: `git checkout -b feature/your-feature-name`
2. Make your changes and commit: `git commit -am 'Add your feature'`
3. Push to the branch: `git push origin feature/your-feature-name`
4. Open a Pull Request on GitHub

### Common Development Tasks

**Adding a new component:**
```bash
# Create component file in src/components/
touch src/components/YourComponent.jsx
```

**Adding new dependencies:**
```bash
npm install package-name
```

**Removing unused dependencies:**
```bash
npm prune
```

## 🤝 Contributing

We welcome contributions from developers, researchers, educators, and innovators of all levels!

### How to Contribute

1. **Fork the repository** on GitHub
2. **Create a feature branch**:
   ```bash
   git checkout -b feature/amazing-feature
   ```
3. **Commit your changes** with descriptive messages:
   ```bash
   git commit -m 'Add some amazing feature'
   ```
4. **Push to your branch**:
   ```bash
   git push origin feature/amazing-feature
   ```
5. **Open a Pull Request** with a clear description of changes

### Contribution Guidelines

- Follow the existing code style and conventions
- Write clear, descriptive commit messages
- Update documentation for any new features
- Ensure all tests pass before submitting a PR
- Be respectful and constructive in discussions

### Areas for Contribution

- 🐛 **Bug Fixes**: Identify and fix issues
- ✨ **New Features**: Implement requested functionality
- 📚 **Documentation**: Improve README, guides, and comments
- 🧪 **Testing**: Add test coverage and improve quality
- 🎨 **UI/UX**: Enhance visual design and usability
- 🌐 **Localization**: Support additional languages
- ♿ **Accessibility**: Improve WCAG compliance

### Reporting Issues

Found a bug or have a suggestion? 

**[Open an Issue](https://github.com/PhilipKone/Kone-Academy/issues)**

Please include:
- A clear, descriptive title
- Detailed description of the issue
- Steps to reproduce (for bugs)
